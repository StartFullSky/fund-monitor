## 告警规则

| 类型 | 条件 | 示例 |
|---|---|---|
| 大幅波动 | 板块内涨跌 ≥ 3%，且比上次记录多 0.5% | 📈 大涨 +3.50% 英伟达 |
| 涨停/跌停 | 涨跌 ≥ 9.9% | 🔴 涨停！中芯国际 |
| 突破5% | 从 <5% 突破到 ≥5% | 📈 突破 +5% |

## Cron 配置

```yaml
schedule: "*/5 9-15 * * 1-5"   # 周一~五 9:00-15:59 每5分钟
no_agent: true
script: "fund_monitor.py"
deliver: "origin"
```

## 数据源

- A股行情: 腾讯 `qt.gtimg.cn`（快、稳、免费）
- 美股行情: 暂不支持（美股非交易时段自动跳过）
- 推送: Bark POST JSON

## 自定义

### 修改告警阈值
编辑 `fund_monitor.py` 中 `SECTORS` 字典的 `alert_pct` 字段。

### 添加/删除板块
编辑 `SECTORS` 字典，按格式添加新板块。

### 添加/删除基金
编辑 `my_funds.json` 和 `SECTORS` 中的 `funds` 映射。

## Pitfalls

### Cron 输出中文乱码
Cron 系统用 GBK 编码显示 UTF-8 中文，导致 `"非交易时间，跳过"` 变成 `闈炰氦鏄撴椂闂达紝璺宠繃`。**解决方案**：cron 脚本的 `print()` 输出必须用英文或 ASCII，避免中文。例如：
```python
# 错误
print("非交易时间，跳过")
# 正确
print("skip")
```

### Terminal 工具会掩码密钥
Terminal 工具会自动掩码 token、密码等敏感信息（如 `ghp_xxxx` 变成 `ghp_l3...74LT`）。**解决方案**：
1. 把 token 写入临时文件（`.token.txt`）
2. 用 curl 或 python 从文件读取 token
3. 用完立即删除临时文件

## 注意事项

- 美股板块在 A 股交易时段会跳过（美股未开盘）
- 港股板块暂未实现（需要港股行情接口）
- 状态文件记录上次涨跌幅，避免重复告警
- 冷却机制：同一股票不会连续告警，需有新的异动才会推送
- **Cron 输出编码**：Windows 上中文输出会变乱码，用英文或静默退出
- **GitHub 推送**：用户希望同时推送 SKILL.md 到 GitHub，不只是 Python 脚本
